# 값 복사, 1부

이전 챕터에서 우리는 소유권과 빌림을 소개했습니다.
특히, 우리는 다음을 명시했습니다:

- Rust의 모든 값은 주어진 시간에 단일 소유자를 가집니다.
- 함수가 값의 소유권을 가져가면("소비하면"), 호출자는 더 이상 해당 값을 사용할 수 없습니다.

이러한 제한은 다소 제한적일 수 있습니다.
때로는 값의 소유권을 가져가는 함수를 호출해야 하지만, 그 후에도 해당 값을 계속 사용해야 할 수도 있습니다.

```rust
fn consumer(s: String) { /* */ }

fn example() {
     let mut s = String::from("hello");
     consumer(s);
     s.push_str(", world!"); // 오류: 이동 후 여기서 값이 빌려짐
}
```

이때 `Clone`이 등장합니다.

## `Clone`

`Clone`은 Rust의 표준 라이브러리에 정의된 트레이트입니다:

```rust
pub trait Clone {
    fn clone(&self) -> Self;
}
```

그 메소드인 `clone`은 `self`에 대한 참조를 받아 동일한 타입의 새로운 **소유된** 인스턴스를 반환합니다.

## 실제 사용

위의 예제로 돌아가서, `consumer`를 호출하기 전에 `clone`을 사용하여 새로운 `String` 인스턴스를 만들 수 있습니다:

```rust
fn consumer(s: String) { /* */ }

fn example() {
     let mut s = String::from("hello");
     let t = s.clone();
     consumer(t);
     s.push_str(", world!"); // 오류 없음
}
```

`s`의 소유권을 `consumer`에게 주는 대신, (`s`를 복제하여) 새로운 `String`을 만들고 그것을 `consumer`에게 줍니다.
`s`는 `consumer` 호출 후에도 유효하고 사용할 수 있습니다.

## 메모리에서

위의 예에서 메모리에서 무슨 일이 일어났는지 살펴봅시다.
`let mut s = String::from("hello");`가 실행되면 메모리는 다음과 같이 보입니다:

```text
                    s
      +---------+--------+----------+
스택  | 포인터  | 길이   | 용량     | 
      |  |      |   5    |    5     |
      +--|------+--------+----------+
         |
         |
         v
       +---+---+---+---+---+
힙:    | H | e | l | l | o |
       +---+---+---+---+---+
```

`let t = s.clone()`이 실행되면, 데이터의 복사본을 저장하기 위해 힙에 완전히 새로운 영역이 할당됩니다:

```text
                    s                                    t
      +---------+--------+----------+      +---------+--------+----------+
스택  | 포인터  | 길이   | 용량     |      | 포인터  | 길이   | 용량     |
      |  |      |   5    |    5     |      |  |      |   5    |    5     |
      +--|------+--------+----------+      +--|------+--------+----------+
         |                                    |
         |                                    |
         v                                    v
       +---+---+---+---+---+                +---+---+---+---+---+
힙:    | H | e | l | l | o |                | H | e | l | l | o |
       +---+---+---+---+---+                +---+---+---+---+---+
```

Java와 같은 언어에서 왔다면, `clone`을 객체의 깊은 복사본을 만드는 방법으로 생각할 수 있습니다.

## `Clone` 구현하기

타입을 `Clone` 가능하게 만들려면 해당 타입에 대해 `Clone` 트레이트를 구현해야 합니다.
거의 항상 derive를 사용하여 `Clone`을 구현합니다:

```rust
#[derive(Clone)]
struct MyType {
    // 필드들
}
```

컴파일러는 예상대로 `MyType`에 대해 `Clone`을 구현합니다: `MyType`의 각 필드를 개별적으로 복제한 다음 복제된 필드를 사용하여 새로운 `MyType` 인스턴스를 구성합니다.
`derive` 매크로에 의해 생성된 코드를 탐색하기 위해 `cargo expand`(또는 IDE)를 사용할 수 있다는 것을 기억하십시오.
