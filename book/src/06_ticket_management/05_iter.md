# `.iter()`

`IntoIterator`는 반복자를 생성하기 위해 `self`를 **소비**합니다.

이것은 장점이 있습니다: 반복자에서 **소유된** 값을 얻습니다.
예를 들어: `Vec<Ticket>`에서 `.into_iter()`를 호출하면 `Ticket` 값을 반환하는 반복자를 얻게 됩니다.

이것은 또한 단점입니다: `into_iter()`를 호출한 후에는 원본 컬렉션을 더 이상 사용할 수 없습니다.
컬렉션을 소비하지 않고, 대신 값에 대한 **참조**를 보면서 컬렉션을 반복하고 싶은 경우가 종종 있습니다.
`Vec<Ticket>`의 경우, `&Ticket` 값을 반복하고 싶을 것입니다.

대부분의 컬렉션은 컬렉션 요소에 대한 참조를 반환하는 `.iter()`라는 메소드를 노출합니다.
예를 들어:

```rust
let numbers: Vec<u32> = vec![1, 2];
// 여기서 `n`은 `&u32` 타입입니다
for n in numbers.iter() {
    // [...]
}
```

이 패턴은 **컬렉션에 대한 참조**에 대해 `IntoIterator`를 구현하여 단순화할 수 있습니다.
위의 예에서 `&Vec<Ticket>`이 될 것입니다.
표준 라이브러리는 이 작업을 수행하며, 이것이 다음 코드가 작동하는 이유입니다:

```rust
let numbers: Vec<u32> = vec![1, 2];
// 여기서 `n`은 `&u32` 타입입니다
// 명시적으로 `.iter()`를 호출할 필요가 없었습니다
// `for` 루프에서 `&numbers`를 사용하는 것으로 충분했습니다
for n in &numbers {
    // [...]
}
```

두 가지 옵션을 모두 제공하는 것이 관용적입니다:

- 컬렉션에 대한 참조에 대한 `IntoIterator` 구현.
- 컬렉션 요소에 대한 참조를 반환하는 `.iter()` 메소드.

전자는 `for` 루프에서 편리하고, 후자는 더 명시적이며 다른 컨텍스트에서 사용할 수 있습니다.
